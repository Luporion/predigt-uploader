# Entwicklungsbericht: CLI-Wizard V1

## Ziel

Den CLI-Wizard fuer Phase 1 nutzerfreundlicher und robuster machen, ohne Vimeo- oder WordPress-Automatisierung einzubauen.

## Geänderte Dateien

- `src/predigt_uploader/cli.py`
- `tests/test_cli.py`
- `docs/dev-log/2026-05-18_1818_cli-wizard-v1.md`

## Was wurde umgesetzt?

- Deutsche CLI-Texte konkreter formuliert.
- Pflichtfelder fragen bei leerer Eingabe erneut nach.
- Ja/Nein-Fragen akzeptieren nur klare Antworten.
- MP4-Pfad wird wiederholt geprüft: leer, nicht gefunden, Ordner statt Datei und falsche Dateiendung.
- Zielordner-Prüfung zeigt Jahresordner, Datum und geplanten Ordnernamen verständlicher an.
- Vorhandene Ordner werden nicht mehr automatisch verwendet, sondern klar bestätigt.
- Wenn ein vorhandener Ordner abgelehnt wird, kann eine Besonderheit fuer einen neuen Ordner eingegeben werden.
- Bei neuer Besonderheit wird sichtbar angezeigt, welcher Ordnername entsteht.
- Keine Vimeo- oder WordPress-Automatisierung und keine Zugangsdaten ergänzt.

## Tests

- `pytest` war nicht direkt im PATH verfügbar.
- `python -m pytest` scheiterte innerhalb der Sandbox am System-Tempordner.
- Nach Freigabe außerhalb der Sandbox erfolgreich ausgeführt: `14 passed in 0.07s`.

## Offene Punkte / Risiken

- `STATUS.md` wurde angefordert, existiert im Repository aber aktuell nicht.
- Der Wizard bleibt ein Terminal-Prototyp; echte Datei-Auswahldialoge sind weiter Phase 2.
